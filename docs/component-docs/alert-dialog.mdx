# Alert Dialog

The Alert Dialog component is a molecule in the AliveUI component library. It's used to interrupt the user with important content and require confirmation or acknowledgement.

## Usage

```tsx
import {
  AlertDialog,
  AlertDialogAction,
  AlertDialogCancel,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogFooter,
  AlertDialogHeader,
  AlertDialogTitle,
  AlertDialogTrigger,
} from "@aliveui/ui";

export default function AlertDialogExample() {
  return (
    <AlertDialog>
      <AlertDialogTrigger>Open</AlertDialogTrigger>
      <AlertDialogContent>
        <AlertDialogHeader>
          <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>
          <AlertDialogDescription>
            This action cannot be undone. This will permanently delete your
            account and remove your data from our servers.
          </AlertDialogDescription>
        </AlertDialogHeader>
        <AlertDialogFooter>
          <AlertDialogCancel>Cancel</AlertDialogCancel>
          <AlertDialogAction>Continue</AlertDialogAction>
        </AlertDialogFooter>
      </AlertDialogContent>
    </AlertDialog>
  );
}
```

## API Reference

The Alert Dialog component is built on top of the Radix UI Alert Dialog primitive and consists of several parts.

### AlertDialog

The root component that contains all the parts of an alert dialog.

| Property       | Type       | Default | Description                                                    |
| -------------- | ---------- | ------- | -------------------------------------------------------------- |
| `defaultOpen`  | `boolean`  | `false` | The open state of the dialog when it is initially rendered     |
| `open`         | `boolean`  | -       | The controlled open state of the dialog                        |
| `onOpenChange` | `function` | -       | Event handler called when the open state of the dialog changes |

### AlertDialogTrigger

The button that opens the dialog.

### AlertDialogContent

The component that contains the content for the dialog.

### AlertDialogHeader

The component that contains the title and description of the dialog.

### AlertDialogFooter

The component that contains the actions for the dialog.

### AlertDialogTitle

The title of the dialog that is announced by screen readers.

### AlertDialogDescription

The description of the dialog that is announced by screen readers.

### AlertDialogAction

The button that confirms the action.

### AlertDialogCancel

The button that cancels the action.

```tsx

      <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>

  </AlertDialogContent>

```

### Controlled Alert Dialog

```tsx
import { useState } from "react";
import { Button } from "@aliveui/ui";

function ControlledAlertDialog() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button variant="outline" onClick={() => setOpen(true)}>
        Show Alert Dialog
      </Button>
      <AlertDialog open={open} onOpenChange={setOpen}>
        <AlertDialogContent>
          <AlertDialogHeader>
            <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>
            <AlertDialogDescription>
              This action cannot be undone. This will permanently delete your
              account and remove your data from our servers.
            </AlertDialogDescription>
          </AlertDialogHeader>
          <AlertDialogFooter>
            <AlertDialogCancel>Cancel</AlertDialogCancel>
            <AlertDialogAction>Continue</AlertDialogAction>
          </AlertDialogFooter>
        </AlertDialogContent>
      </AlertDialog>
    </>
  );
}
```

## Accessibility

The Alert Dialog component follows accessibility best practices:

- Uses Radix UI's accessible alert dialog primitive
- Alert dialogs are announced properly to screen readers
- Users are prevented from interacting with other content while the alert dialog is open
- Focus is automatically trapped within the dialog and restored to the trigger when closed
- The escape key is supported for closing the dialog
- Properly implements the `alertdialog` role
- Properly implements `aria-labelledby` and `aria-describedby` for screen readers

## Implementation Details

The Alert Dialog component is built on top of Radix UI's Alert Dialog primitive, ensuring robust accessibility and consistent behavior across browsers. It's styled with Tailwind CSS and is fully customizable through className props. The component includes animations for opening and closing the dialog, and is designed to be responsive across different screen sizes.
