# Hover Card

The Hover Card component is a molecule in the AliveUI component library. It displays additional information in a floating card when a user hovers over a trigger element.

## Usage

```tsx
import { HoverCard, HoverCardTrigger, HoverCardContent } from "@aliveui/ui";
import { Avatar } from "@aliveui/ui";

export function HoverCardDemo() {
  return (
    <HoverCard>
      <HoverCardTrigger asChild>
        <a href="#" className="text-blue-500 underline">
          @johndoe
        </a>
      </HoverCardTrigger>
      <HoverCardContent className="w-80">
        <div className="flex justify-between space-x-4">
          <Avatar src="https://github.com/johndoe.png" fallback="JD" />
          <div className="space-y-1">
            <h4 className="text-sm font-semibold">@johndoe</h4>
            <p className="text-sm">UI/UX designer and full-stack developer</p>
            <div className="flex items-center pt-2">
              <span className="text-xs text-muted-foreground">
                Joined December 2023
              </span>
            </div>
          </div>
        </div>
      </HoverCardContent>
    </HoverCard>
  );
}
```

## API Reference

The Hover Card component is built on top of Radix UI's Hover Card primitive.

### HoverCard

The root component that wraps the hover card trigger and content.

| Property       | Type       | Default | Description                                    |
| -------------- | ---------- | ------- | ---------------------------------------------- |
| `defaultOpen`  | `boolean`  | -       | The default open state when initially rendered |
| `open`         | `boolean`  | -       | The controlled open state of the hover card    |
| `onOpenChange` | `function` | -       | Event handler called when open state changes   |
| `openDelay`    | `number`   | `700`   | The delay in ms before the hover card opens    |
| `closeDelay`   | `number`   | `300`   | The delay in ms before the hover card closes   |

### HoverCardTrigger

The element that triggers the hover card when hovered. By using `asChild`, it can be rendered as any element.

### HoverCardContent

The component that contains the hover card content.

| Property          | Type                                     | Default    | Description                                                |
| ----------------- | ---------------------------------------- | ---------- | ---------------------------------------------------------- |
| `align`           | `string`                                 | `"center"` | The alignment of the hover card relative to the trigger    |
| `sideOffset`      | `number`                                 | `4`        | The distance in pixels from the trigger                    |
| `alignOffset`     | `number`                                 | -          | The offset in pixels from the aligned edge                 |
| `side`            | `"top" \| "right" \| "bottom" \| "left"` | -          | The preferred side of the trigger to render the hover card |
| `avoidCollisions` | `boolean`                                | `true`     | Whether to avoid collisions with boundaries                |

## Examples

### Basic Hover Card

```tsx
<HoverCard>
  <HoverCardTrigger asChild>
    <Button variant="link">Hover over me</Button>
  </HoverCardTrigger>
  <HoverCardContent>
    <div className="space-y-2">
      <h4 className="text-sm font-semibold">Information</h4>
      <p className="text-sm">
        This is a simple hover card that displays additional information on
        hover.
      </p>
    </div>
  </HoverCardContent>
</HoverCard>
```

## Accessibility

The Hover Card component follows these accessibility best practices:

- Proper keyboard focus management
- Appropriate ARIA attributes for screen readers
- Adheres to WAI-ARIA design patterns
- Timed delays to prevent accidental triggering
- Mouse and touch events are handled appropriately

## Implementation Details

The Hover Card component is built on top of Radix UI's Hover Card primitive, providing a robust and accessible implementation. It is styled with Tailwind CSS and includes the following features:

- Customizable open and close delays
- Flexible positioning options
- Seamless integration with other AliveUI components
- Smooth animations for opening and closing
- Responsive design that works across different device sizes

Hover cards are ideal for providing additional context without requiring a click or taking the user away from their current task. They're commonly used for user profiles, tooltips, preview cards, and contextual help.
