# Scroll Area

The Scroll Area component is an atom in the AliveUI component library. It's used to create a scrollable container with custom-styled scrollbars.

## Usage

```tsx
import { ScrollArea } from "@aliveui/ui";

export default function ScrollAreaExample() {
  return (
    <ScrollArea className="h-[200px] w-[350px] rounded-md border p-4">
      <div>
        <p>Long content that will cause scrolling...</p>
        {/* Add lengthy content here */}
      </div>
    </ScrollArea>
  );
}
```

## API Reference

The Scroll Area component is built on top of the Radix UI Scroll Area primitive.

### ScrollArea

| Property          | Type                                        | Default   | Description                                                                         |
| ----------------- | ------------------------------------------- | --------- | ----------------------------------------------------------------------------------- |
| `type`            | `"auto" \| "always" \| "scroll" \| "hover"` | `"hover"` | Describes the nature of scrollbar visibility                                        |
| `scrollHideDelay` | `number`                                    | `600`     | Delay in milliseconds before the scrollbar is hidden after the user stops scrolling |
| `dir`             | `"ltr" \| "rtl"`                            | -         | The reading direction of the scroll area                                            |
| `asChild`         | `boolean`                                   | `false`   | When `true`, the component will render as its child                                 |

### ScrollBar

| Property      | Type                         | Default      | Description                        |
| ------------- | ---------------------------- | ------------ | ---------------------------------- |
| `orientation` | `"horizontal" \| "vertical"` | `"vertical"` | The orientation of the scrollbar   |
| `forceMount`  | `boolean`                    | `false`      | Forces the scrollbar to be visible |

## Examples

### Basic Scroll Area

```tsx
<ScrollArea className="h-[200px] w-[350px] rounded-md border p-4">
  <div>
    <h4 className="mb-4 text-sm font-medium leading-none">Tags</h4>
    {Array.from({ length: 50 }).map((_, i) => (
      <div key={i} className="text-sm">
        Tag {i + 1}
      </div>
    ))}
  </div>
</ScrollArea>
```

### Scroll Area with Custom Style

```tsx
<ScrollArea className="h-72 w-48 rounded-md border">
  <div className="p-4">
    <h4 className="mb-4 text-sm font-medium leading-none">Channels</h4>
    {Array.from({ length: 30 }).map((_, i) => (
      <div
        key={i}
        className="text-sm my-2 flex items-center gap-2 rounded-md p-2 hover:bg-muted/50 cursor-pointer"
      >
        <div className="h-2 w-2 rounded-full bg-primary" />
        <span>Channel {i + 1}</span>
      </div>
    ))}
  </div>
</ScrollArea>
```

### Horizontal Scroll Area

```tsx
<ScrollArea className="w-96 whitespace-nowrap rounded-md border">

        <div className="text-sm text-muted-foreground">

        </div>
      </div>
    ))}
  </div>
</ScrollArea>
```

## Accessibility

The Scroll Area component follows accessibility best practices:

- Uses Radix UI's accessible scroll area primitive
- Automatically hides scrollbars when not in use, but keeps them accessible via keyboard
- Provides a native scrolling experience that respects the user's system preferences
- Scrollbars are visible when needed and don't interfere with content
- Maintains keyboard navigation compatibility

## Implementation Details

The Scroll Area component is built on top of Radix UI's Scroll Area primitive, ensuring robust accessibility and consistent behavior across browsers. It's styled with Tailwind CSS and designed to provide a custom scrollbar that's visually consistent with the rest of the UI while maintaining native scrolling behavior.
