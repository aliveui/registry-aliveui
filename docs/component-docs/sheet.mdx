# Sheet

The Sheet component is a molecule in the AliveUI component library. It provides a side-anchored panel that slides in from the edge of the screen, similar to a drawer or sidebar, allowing for additional content or controls.

## Usage

```tsx
import {
  Sheet,
  SheetContent,
  SheetDescription,
  SheetHeader,
  SheetTitle,
  SheetTrigger,
} from "@aliveui/ui";

export function SheetDemo() {
  return (
    <Sheet>
      <SheetTrigger>Open Sheet</SheetTrigger>
      <SheetContent>
        <SheetHeader>
          <SheetTitle>Sheet Title</SheetTitle>
          <SheetDescription>
            This is a description for the sheet.
          </SheetDescription>
        </SheetHeader>
        <div className="py-4">{/* Your sheet content here */}</div>
      </SheetContent>
    </Sheet>
  );
}
```

## API Reference

### Sheet

The root component that manages the state of the sheet.

```tsx
<Sheet>{/* SheetTrigger and SheetContent */}</Sheet>
```

### SheetTrigger

The button that opens the sheet when clicked.

```tsx
<SheetTrigger>Open Sheet</SheetTrigger>
```

### SheetContent

The content of the sheet that appears when it's opened. Supports different sides for the sheet to appear from.

```tsx
<SheetContent side="right">{/* Sheet content */}</SheetContent>
```

**Props**

- `side`: The side from which the sheet will appear. Options: `"top"`, `"right"`, `"bottom"`, `"left"`. Default: `"right"`.

### SheetHeader

A container for the sheet's header content, typically containing the title and description.

```tsx
<SheetHeader>
  <SheetTitle>Sheet Title</SheetTitle>
  <SheetDescription>Sheet description</SheetDescription>
</SheetHeader>
```

### SheetTitle

The title of the sheet.

```tsx
<SheetTitle>Sheet Title</SheetTitle>
```

### SheetDescription

A description for the sheet, typically explaining its purpose.

```tsx
<SheetDescription>This is a description for the sheet.</SheetDescription>
```

### SheetFooter

A container for the sheet's footer content, typically containing action buttons.

```tsx
<SheetFooter>
  <Button>Action</Button>
</SheetFooter>
```

### SheetClose

A component that can be used to close the sheet when clicked.

```tsx
<SheetClose>
  <Button>Close</Button>
</SheetClose>
```

## Accessibility

The Sheet component is built on top of Radix UI's Dialog primitive, ensuring excellent accessibility:

- Follows the [WAI-ARIA Dialog Modal Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/dialogmodal/)
- Focus is automatically trapped within the sheet when opened
- Proper ARIA attributes and roles for screen readers
- Keyboard navigation - close with Escape key
- Click-outside behavior to dismiss
- Focus is returned to the trigger when the sheet is closed

## Implementation Details

The Sheet component is built using Radix UI's Dialog primitive and Tailwind CSS for styling. It provides a versatile way to present secondary content or controls that don't need to occupy the entire screen.

The component supports:

- Multiple positions (top, right, bottom, left)
- Smooth animations
- Backdrop blur
- Responsive widths
- Focus management
- Keyboard interaction

The implementation can be used for a variety of use cases such as navigation menus, settings panels, forms, and detail views.
