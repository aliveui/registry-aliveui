# Switch

The Switch component is an atom in the AliveUI component library. It's used to toggle between an on and off state.

## Usage

```tsx
import { Switch } from "@aliveui/ui";
import { Label } from "@aliveui/ui";

export default function SwitchExample() {
  return (
    <div className="flex items-center space-x-2">
      <Switch id="airplane-mode" />
      <Label htmlFor="airplane-mode">Airplane Mode</Label>
    </div>
  );
}
```

## API Reference

The Switch component is built on top of the Radix UI Switch primitive.

### Properties

| Property          | Type       | Default | Description                                                                                 |
| ----------------- | ---------- | ------- | ------------------------------------------------------------------------------------------- |
| `checked`         | `boolean`  | -       | The controlled checked state of the switch                                                  |
| `defaultChecked`  | `boolean`  | -       | The default checked state when initially rendered                                           |
| `onCheckedChange` | `function` | -       | Event handler called when the checked state changes                                         |
| `disabled`        | `boolean`  | `false` | When `true`, prevents the user from interacting with the switch                             |
| `required`        | `boolean`  | `false` | When `true`, indicates that the user must check the switch before the form can be submitted |
| `name`            | `string`   | -       | The name of the switch. Submitted with its owning form as part of a name/value pair         |
| `value`           | `string`   | -       | The value given as data when submitted with a name                                          |

## Examples

### Basic Switch

```tsx
<div className="flex items-center space-x-2">
  <Switch id="airplane-mode" />
  <Label htmlFor="airplane-mode">Airplane Mode</Label>
</div>
```

### Checked by Default

```tsx
<div className="flex items-center space-x-2">
  <Switch id="wifi" defaultChecked />
  <Label htmlFor="wifi">Wi-Fi</Label>
</div>
```

### Disabled Switch

```tsx
<div className="flex items-center space-x-2">
  <Switch id="bluetooth" disabled />
  <Label htmlFor="bluetooth" className="text-muted-foreground">
    Bluetooth
  </Label>
</div>
```

### Controlled Switch

```tsx
import { useState } from "react";

function ControlledSwitch() {
  const [checked, setChecked] = useState(false);

  return (
    <div className="space-y-2">
      <div className="flex items-center space-x-2">
        <Switch
          id="controlled-switch"
          checked={checked}
          onCheckedChange={setChecked}
        />
        <Label htmlFor="controlled-switch">{checked ? "On" : "Off"}</Label>
      </div>
      <p className="text-sm text-muted-foreground">
        The switch is {checked ? "on" : "off"}.
      </p>
    </div>
  );
}
```

### Form Integration

```tsx
<form
  onSubmit={(e) => {
    e.preventDefault();
    const formData = new FormData(e.target);
    const enabled = formData.get("notifications") === "on";
    console.log(`Notifications: ${enabled ? "Enabled" : "Disabled"}`);
  }}
>
  <div className="flex items-center space-x-2">
    <Switch id="notifications" name="notifications" />
    <Label htmlFor="notifications">Enable notifications</Label>
  </div>
  <Button type="submit" className="mt-4">
    Save preferences
  </Button>
</form>
```

## Accessibility

The Switch component follows accessibility best practices:

- Uses Radix UI's accessible switch primitive
- Supports keyboard navigation (Tab to focus, Space to toggle)
- Properly communicates state changes to screen readers
- Supports form association
- Provides proper ARIA attributes
- Maintains sufficient color contrast

## Implementation Details

The Switch component is built on top of Radix UI's Switch primitive, ensuring robust accessibility and consistent behavior across browsers. It's styled with Tailwind CSS and is fully customizable through className props.
